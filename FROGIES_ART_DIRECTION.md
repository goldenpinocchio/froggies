# Froggies Art Direction Spec

## North star
Froggies should feel like a **cute-but-not-mascot** pond collection with a soft editorial edge:
- target supply: **2,016**
- one frog hero per NFT
- trait combinations should stay deterministic and easy to explain
- **single frog hero per NFT**
- **modular traits** that can be swapped cleanly
- **flat vector / poster-like** illustration language
- **pastel-meets-pond** palette with a calm, lo-fi mood
- **deterministic trait logic** driven by Bitcoin block data

The attached 6-panel reference is **not** the final output format. It is a **variation board** that shows how one base scene can branch into multiple versions. The actual collection should still mint as **one image per NFT**, assembled from the same trait stack.

## Visual target
Think:
- editorial pond scene
- mid-century restraint
- lo-fi chill atmosphere
- cute, soft, readable at thumbnail size
- simple geometry and clean silhouettes
- enough personality to feel charming, not plush or toy-like

Avoid:
- blocky / chunky / pixel-like frog shapes
- mascot energy
- overly plush roundness
- sticker-style framing
- noisy backgrounds that fight the frog
- overworked facial detail

## Core composition
The base scene should always read in this order:
1. **Background mood** — sky/water wash, subtle light shape, or simple pond gradient
2. **Pond surface** — calm water, ripples, or a soft reflective base
3. **Hero frog** — central subject, clearly readable as a frog
4. **Swamp/pond accessories** — lily pads, lotus, reeds, coins, flies, bubbles, etc.
5. **Head gear / expression details** — small but legible accent traits

The frog should stay the visual anchor. Background and accessories should support the frog, not dominate it.

## Base style language
- Flat vector shapes
- Soft curves, but with clearer frog anatomy than a blob
- Gentle contrast between frog and water
- Muted pastel palette with a few richer accents
- Thin, deliberate outlines or edge separation if needed
- Editorial spacing: leave room around the character
- Calm, composed pose rather than action-heavy staging

## Trait architecture
Keep the collection organized around the user’s trait buckets:

### 1) Body color
This is the most important base trait. It should set the overall mood of the frog without breaking the scene.

Good direction:
- soft greens
- sage / mint
- pale jade
- moss / olive-muted variants
- occasional warmer or cooler pastel outliers

Rule:
- body color should change the feel of the frog, but not the design language
- even rare colors should still feel like the same Froggies family

### 2) Eyes
Eyes should carry personality more than complexity.

Good directions:
- round and calm
- slightly sleepy / half-lidded
- glossy / reflective
- curious / wide
- rare special states like laser or intense glow can exist later as overlays

Rule:
- eyes should remain readable at small size
- expression should come from the eye shape + highlight, not just extra decoration

### 3) Mouth / expression
The mouth should keep the frog cute and emotionally legible.

Good directions:
- tiny smile
- neutral content face
- soft grin
- surprised / puckered
- sleepy / relaxed

Rule:
- expression should feel subtle and collectible, not cartoonishly loud

### 4) Head gear
Head gear should read like a small style accent, not a costume.

Good directions:
- lily crown
- tiny cap
- simple headband
- leaf wrap
- small ceremonial / pond-themed detail

Rule:
- head gear must never overpower the frog silhouette
- keep it small, graphic, and easy to swap in/out

### 5) Swamp accessories
These are the fun overlay traits that give each NFT a little personality and scene texture.

Good directions:
- lily pad props
- lotus details
- floating leaf bits
- tiny fly / bug companion
- bubbles
- coins / sparkles / pond debris
- subtle water ripples or splash marks

Rule:
- accessories should feel like part of the pond world, not random add-ons
- use them to create rarity without breaking the base composition

### 6) Background
Background should set the mood and help differentiate versions without stealing focus.

Good directions:
- light / dusk / night pond atmospheres
- soft color-field backgrounds
- simple sun or moon shape
- subtle gradients or poster blocks
- variation in water tone, sky tone, or horizon treatment

Rule:
- background should remain simple enough to keep the frog hero dominant

## Recommended base direction
If we need one strongest base to build from, it should be this:

**Editorial Pond**
- calm flat-vector pond scene
- soft pastel body colors
- frog with clearer anatomical shape
- readable eyes and subtle mouth expression
- minimal head gear
- small pond accessories
- background with enough atmosphere to feel art-directed, not empty

This is the best default because it balances:
- cute
- collectible
- modular
- clean
- scalable for a trait system

## Variation strategy
Use the reference-style 6-up board as a **design lab**, not the final product.

Each panel should vary one or two things at a time, such as:
- body color
- eye expression
- head gear
- swamp accessory mix
- background mood

That makes it easy to judge which combinations feel strongest before finalizing the trait library.

## Deterministic trait logic
The final trait system should still be driven by Bitcoin block data.

Principles:
- common block outcomes should map to common traits
- rarer block outcomes should map to rarer traits
- special traits like gold / diamond / laser-eyes should be explicit rarities, not default decoration
- the mapping should be easy to explain and easy to copy/paste into the launch docs

## What success looks like
A strong Froggies base should:
- read instantly as a frog
- feel cute without becoming mascoty
- support a modular trait system
- look good at thumbnail size
- hold up as a single-image NFT
- feel like a finished art direction, not a rough sketch
